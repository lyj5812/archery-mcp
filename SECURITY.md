# 安全策略

## 支持范围

安全修复应用于项目的最新版本。

## 报告安全问题

请使用 GitHub 仓库的私密漏洞报告功能。不要创建包含账号密码、生产地址、SQL 结果或漏洞利用细节的公开 Issue。

## 部署建议

- 使用专用的 Archery 最小权限服务账号。
- 优先使用只读数据库实例。
- 保持 TLS 证书校验开启。
- 保持 Archery 的查询权限、超时、审计日志、行数限制和数据脱敏功能开启。
- 通过进程环境或密钥管理服务提供凭据，不要写入仓库文件。
- MCP 工具参数可能包含 SQL 或对象标识，需要审查 MCP 客户端的日志保存策略。
- CSV 导出目录可能包含敏感数据，需要限制文件系统权限并设置清理周期。

本地 SQL 校验仅用于纵深防御，不能替代数据库权限或 Archery 授权。
